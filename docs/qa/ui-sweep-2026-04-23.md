## calculator (Kalkulator Keuangan) — REGRESSION UTAMA

🔴 calculator:Bandingkan Kota desktop 1327px — **tab container width berubah** antar sub-tab:
  - "Perencanaan Budget" aktif: tablist width = **775 px** (auto-fit parent).
  - "Info Gaji" aktif: tablist width = **775 px** (sama).
  - "Bandingkan Kota" aktif: tablist width = **481 px** (menciut ~294 px).

  Root cause terverifikasi via DOM inspection: `space-y-6 > [role="tablist"]` berada di dalam wrapper `<div class="max-w-6xl mx-auto px-4 sm:px-6 lg:px-8 py-8">`. Pada sub-page Bandingkan Kota, konten di dalamnya tidak punya child yang meng-stretch max-w-6xl sehingga intrinsic content-width (≈545 px) yang kepakai, bukan 1063 px dari parent flex-col (align-items default = normal/stretch di flex-col, tapi block container di dalam flex item tidak auto-stretch karena tidak ada min-width: 0 + konten intrinsic).

  Kontras dengan Perencanaan Budget & Info Gaji: di sana ada grid utama 2-kolom (form kiri + ringkasan kanan) yang full-width, jadi `max-w-6xl` ter-stretch ke 839 px. Inilah kenapa tablist ikut stretching ke 775 px.

  Fix (pilih salah satu, rekomendasi #1):
  1. **Stretch max-w-6xl wrapper**: tambahkan `w-full` ke class `max-w-6xl mx-auto ...` — jadi `w-full max-w-6xl mx-auto ...`. Ini memaksa wrapper selalu mengisi parent flex item sampai max-w-6xl terpenuhi, independen isi. Paling murah dan paling safe.
  2. Atau, pindah tablist ke level layout lebih luar sehingga dia tidak tergantung konten sub-page.
  3. Atau, migrate semua tab ke `MagneticTabs` yang punya width fixed/fit-content controlled.

🟡 calculator:TabActiveState — active tab di "Perencanaan Budget" punya bold font-weight persepsi, sementara saat first render tampak semi-bold. `computedStyle` konsisten `font-weight: 500` (semua tab); perbedaan visual berasal dari background putih solid vs transparan di active state + subtle shadow. Pastikan design token konsisten (fine as-is, noted).

🔵 calculator:TablistClassName — DOM saat navigasi pertama kali setelah dark mode toggle menunjukkan tablist punya dua set class berbeda bergantung render:
  - Versi A: `h-9 items-center justify-center rounded-lg bg-muted p-1 text-muted-foreground grid w-full max-w-lg grid-cols-3` (512px)
  - Versi B: `h-9 grid w-full items-center gap-1 overflow-x-auto rounded-lg bg-muted p-1 ... auto-cols-fr ... grid-cols-3` (mengisi parent)

  Artinya ada **dua implementasi Tabs** yang berbeda untuk halaman yang sama — kemungkinan dari dynamic import/chunk yang di-split. Fix: pastikan satu satu sumber `<Tabs>` wrapper dipakai; cek apakah ada custom wrapper di `/dashboard/calculator/*.tsx` yang override class.

🔵 calculator:PengeluaranBulanan — slider terdaftar panjang (Tempat Tinggal, Makanan, Transport, Utilitas, Hiburan, Lainnya) dengan value `Rp ...`. Di viewport 1327 px OK; wajib QA mobile untuk layout slider+nilai tidak overlap di 390 px.

🔵 calculator:ThousandSeparator — input "Gaji Bulanan (IDR)" menerima value `15000000` tanpa auto-format jadi `15.000.000`. Namun di label kanan ditampilkan "Rp 15.000.000". Fix: input harus auto-format while typing + strip non-digit supaya konsisten (commit 15563a9 ada tapi terlihat belum terpasang di semua input).

🔵 calculator:RangeSliderLabel — "Rentang slider: Rp 0 – Rp 50.000.000 / bulan" muncul di satu rendering (gaji tab) tapi di first visit tidak muncul. Pastikan label ini konsisten ada.

🔵 calculator:RentangGajiChart — di "Info Gaji" tab, axis Y menampilkan "5000000, 7000000, 3000000, 3000000, 0" — angka tidak berformat separator ID dan ada duplikasi `3000000` yang tampak bug data. Fix: tickFormatter pakai Intl.NumberFormat('id-ID') dan cek data dedup di query.
